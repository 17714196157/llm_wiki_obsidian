# 会议汇报：LLM 推理效率优化 -- Think Deep, Not Just Long

---

## 一、研究背景

### 当前业界做法
主流大模型（OpenAI o1/o3、DeepSeek-R1、Claude 3.7/4、Qwen3）都通过**增加推理长度**来提升推理能力 -- 鼓励模型生成更长的思考链（Chain-of-Thought）。

**核心假设**：想得越多（token 越多），推理越准。

### 问题发现
近年的多篇研究发现这个假设有问题：
- **倒 U 型关系**：CoT 长度与性能之间存在倒 U 型曲线，超过某个点后性能反而下降
- **逆缩放行为**：更长推理系统性降低准确率
- **过度推理（Overthinking）**：模型在长推理中放大错误判断，纠缠于无关细节

**核心问题**：我们缺少一个可靠的指标来衡量模型推理时的"真正思考努力"。

---

## 二、核心贡献

这篇论文提出了两个东西：

1. **Deep-Thinking Ratio (DTR)** -- 一个新的指标，能可靠衡量模型推理时的思考质量
2. **Think@n** -- 一个基于 DTR 的新策略，在保持甚至提升准确率的同时，减少约 50% 推理成本

---

## 三、DTR 指标：实验设计与结果

### 3.1 DTR 是什么？

传统方法看的是"生成了多少 token"（表面特征）。

DTR 看的是"每个 token 的预测在模型内部经历了什么"（机制特征）：
 
- **方法**：把每一层的隐藏状态投影到词汇空间，计算每层预测分布与最终层分布之间的 Jensen-Shannon 散度（JSD）
- **核心假设**：如果一个 token 的预测需要到很深的层才稳定下来，说明模型对它进行了"深度思考"；如果浅层就稳定，说明思考量较少
- **计算**：DTR = 生成序列中"深度思考 token"的比例

**类比**：
- 浅层稳定 = 凭直觉回答（不经过深思熟虑）
- 深层才稳定 = 反复推敲后得出结论（真正在思考）

### 3.2 实验设置

| 维度 | 详情 |
|------|------|
| **模型（8 个变体）** | GPT-OSS-20B（低/中/高推理级别）、GPT-OSS-120B（低/中/高）、DeepSeek-R1-70B、Qwen3-30B-Thinking |
| **基准任务（4 个）** | AIME 2024（数学竞赛）、AIME 2025、HMMT 2025（哈佛-MIT 数学竞赛）、GPQA-diamond（研究生级科学问答） |
| **采样设置** | 每题 25 个响应，30 个随机种子 |
| **超参数** | 收敛阈值 g=0.5，深度分数 ρ=0.85 |

### 3.3 对比基线

论文对比了 6 种基线方法 + DTR：

| 类别 | 方法 | 原理 |
|------|------|------|
| **长度指标** | Token 数量 | 生成了多少 token |
| | 反向 Token 数量 | 越短越好（事后统计调整） |
| **置信度指标** | 对数概率 | 模型对输出的置信度 |
| | 负困惑度 | 生成流畅度 |
| | 负熵 | 预测确定性 |
| | Self-Certainty | 与均匀分布的 KL 散度 |
| **本文方法** | **DTR** | 深度思考 token 比例 |

### 3.4 关键实验结果

**Pearson 相关系数（与准确率的关系）：**

| 方法 | 平均相关系数 | 含义 |
|------|-------------|------|
| Token 数量 | **-0.544** | 负相关 -- 越长越差 |
| 反向 Token 数量 | +0.544 | 事后调整，无机制解释 |
| 对数概率 | +0.668 | 中等正相关 |
| Self-Certainty（最佳基线） | +0.703 | 较强正相关，但不稳定 |
| **DTR（本文）** | **+0.828** | **强正相关，最稳定** |

**具体数据（GPT-OSS-120B-medium）：**

| 基准 | Token 长度 | Self-Certainty | DTR |
|------|-----------|----------------|-----|
| AIME 2025 | -0.289 | +0.803 | +0.930 |
| AIME 2024 | -0.456 | +0.664 | +0.863 |
| HMMT 2025 | -0.748 | +0.779 | +0.866 |
| GPQA-diamond | -0.684 | +0.566 | +0.654 |

**关键结论：**
1. DTR 在所有 32 个（模型 × 基准）配置中，有 30 个呈现正相关
2. 只有 2 个配置出现负相关（基线方法有更多负相关）
3. DTR 在所有模型族（GPT-OSS、DeepSeek-R1、Qwen3）上均有效，证明了跨模型泛化能力

### 3.5 超参数敏感性分析

论文测试了不同收敛阈值（g）和深度分数（ρ）：

- **g=0.3**（太宽松）：相关性变平，无法区分高质量和低质量推理
- **g=0.5~0.7**：稳定的强正相关
- **g=0.9**（太严格）：趋势仍然存在但略不稳定
- **ρ=0.85** 是最佳平衡点
- **结论**：DTR 对超参数相对鲁棒，不需要精细调参

### 3.6 替代距离度量验证

论文还验证了使用 KL 散度和余弦相似度代替 JSD 的效果：
- **JSD（主方法）**：最佳稳定性和相关性
- **KL 散度**：效果接近但不如 JSD 稳定
- **余弦相似度**：效果稍差，但仍优于长度指标

## 四、Think@n 策略：实验设计与结果

### 4.1 问题定义

在 test-time scaling 中，常用方法是**重复采样 + 多数投票**（Self-Consistency），但这对所有样本都完整生成，计算成本高。

**核心问题**：能否在生成过程中就识别出"高质量样本"，提前终止低质量样本？

### 4.2 Think@n 算法

```
1. 并行生成 n 个样本
2. 用前 50 个 token 估计每个样本的 DTR
3. 只保留 DTR 最高的 top-50% 样本
4. 提前终止其余样本（节省计算）
5. 对保留样本的答案进行多数投票
```

### 4.3 对比方法

| 方法 | 原理 | 需要完整生成？ |
|------|------|---------------|
| **Cons@n** | 标准 Self-Consistency，多数投票 | 是 |
| **Mean@n** | 所有样本的平均准确率 | 是 |
| **Long@n** | 投票给最长的 50% 样本 | 是 |
| **Short@n** | 投票给最短的 50% 样本 | 部分（可提前终止） |
| **Self-Certainty@n** | 投票给 Self-Certainty 最高的 50% | 部分 |
| **Think@n** | 投票给 DTR 最高的 50% | 部分 |

### 4.4 实验结果

**GPT-OSS-120B-medium 结果：**

| 方法 | AIME 25 | AIME 24 | HMMT 25 | GPQA-D | 平均成本 | 成本降低 |
|------|---------|---------|---------|--------|----------|----------|
| Cons@n（基线） | 92.7% | 92.7% | 80.0% | 73.8% | 100% | -- |
| Mean@n | 80.0% | 81.6% | 62.6% | 69.9% | 100% | -- |
| Long@n（选最长） | 86.7% | 86.7% | 73.3% | 73.2% | 100% | -- |
| Short@n（选最短） | 87.3% | 88.0% | 77.3% | 73.3% | ~83% | -17% |
| Self-Certainty@n | 87.3% | 91.3% | 78.0% | 76.0% | ~50% | -49% |
| **Think@n** | **94.7%** | **93.3%** | **80.0%** | **74.7%** | **~50%** | **-49%** |

**关键结论：**
1. Think@n 在 4 个基准中有 3 个**超过了**标准 Self-Consistency 的准确率
2. 计算成本减少了约 **49%**
3. Short@n 比 Long@n 表现更好，证明"选短的"比"选长的"更可靠（但仍然不如 Think@n）
4. Self-Certainty@n 虽然成本也减半，但准确率在 3/4 基准上低于 Think@n

**Qwen3-4B-Thinking 结果（小模型验证）：**

| 方法 | AIME 25 | AIME 24 | HMMT 25 | GPQA-D | 成本降低 |
|------|---------|---------|---------|--------|----------|
| Cons@n | 86.7% | 93.3% | 63.3% | 67.8% | -- |
| **Think@n** | **90.0%** | **93.3%** | **66.7%** | **69.7%** | **-50%** |

结论在小模型上同样成立。

### 4.5 前缀长度分析

一个关键问题是：需要多长的前缀才能可靠估计 DTR？

| 前缀长度 | AIME 25 准确率 | 推理成本（k tokens） |
|----------|---------------|---------------------|
| 50 tokens | **94.7%** | 155.4 |
| 100 tokens | 92.0% | 154.1 |
| 500 tokens | 92.7% | 153.2 |
| 1000 tokens | 92.7% | 177.4 |
| 2000 tokens | 92.0% | 198.8 |
| 完整序列 | 94.0% | 307.6 |
| Cons@n（参考） | 90.0% | 307.6 |

**关键发现**：**仅用 50 个 token 的前缀就能达到最佳效果**，比用更长前缀或完整序列都更好，同时大幅降低成本。

## 五、对我们的启示

### 5.1 短期可落地

1. **推理监控升级**：不只监控输出长度，还应引入思考质量指标
2. **多采样 + 筛选策略**：在关键任务上尝试多采样，用置信度或长度等简单指标做初步筛选
3. **设置合理的最大 token 上限**：避免过度推理浪费资源

### 5.2 中长期方向

1. **内部表征监控**：如果需要访问模型内部隐藏状态，可以实现类似 DTR 的指标
2. **自适应推理预算**：根据问题难度动态调整推理深度
3. **训练时优化**：用 DTR 或类似指标作为奖励信号，引导模型学会"有效思考"
4. **成本优化**：在大批量推理场景中，类似 Think@n 的策略可以显著降低推理成本

### 5.3 注意事项

- DTR 需要访问模型的**中间层隐藏状态**，对于闭源 API（如 GPT-4、Claude）可能无法直接实现
- 但对于**自部署开源模型**（如 Llama、Qwen、DeepSeek 的开源版本），完全可以实现
- 论文已在 GPT-OSS（OpenAI 开源模型）、DeepSeek-R1 蒸馏版、Qwen3 上验证
